Wasm Systems Overview

Wasm (or Web Assembly) is a portable binary format perfectly suitable for executing code you write on the web or embedded in another program. Ecsact supports having your system implementations in the Wasm format. System implementations compiled to Wasm are the best all around way for a working with Ecsact.

This article should give you everything you need to know in order to create a valid system implementation .wasm file.

Wasm Imports & Exports

Only certain exports of a .wasm file are used by the ecsact runtime and a limited set of imports are allowed. Valid exports are only functions with the system execution signature. Exports that do not meet that requirement are completely ignored. Imports are much more strict than exports. The only imports that are allowed are ecsact_system_execution_context_* functions from the ecsact runtime.

The system implementation signature in wasm32 would look like this: 


		(func (param i32) (result void) ...)
