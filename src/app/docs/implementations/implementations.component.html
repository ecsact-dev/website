Runtime Implementations

A runtime library does not have an official implementation. The runtime modules are a standard that an implementation must comply with. Which is to say that each implementation behaves the same, but may perform different in terms of execution speed and/or storage size. An implementation may even be tailored to a set of ecsact files. For example, the Ecsact Runtime Builder (RTB) is a utility for creating tailored runtimes for generic implementations.

Building an Implementation

Ecsact is in heavy development so the details for building an implementation is sparse. The best you can do is look at an existing implementation and read the ecsact_runtime reference documentation for how each method should work.

Generic Implementations

A generic runtime implementation is incomplete without Ecsact files or derivatives (code generation) of Ecsact files. Not to be confused with an implementation with a generic runtime configuration.

Known Implementations

  • Ecsact Runtime EnTT

    — A generic implementation maintained by the Ecsact Dev Team and available as one of the implementations in the Ecsact Runtime Builder (RTB). Built with the excellent library open_in_new EnTT.

  • Ecsact Interpret Resolver Runtime

    — A tooling runtime used by the Ecsact Interpreter to evaluate Ecsact files.
