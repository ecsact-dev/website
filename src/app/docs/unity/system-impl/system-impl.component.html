Implementing Systems in Unity

The Ecsact Unity integration provides a convenient way to implement your systems in C# or WebAssembly.

C# System Implementation

To enable C# system implementations open your Unity project and go to Project Settings arrow_right Ecsact arrow_right System Implementations and change Source to C#. In addition to setting the source to C# you must also give a C# Assembly. The specified C# Assembly must contain your C# system implementation functions. If you're not familiar with C# Assemblies and Unity, see Unity's open_in_new Assembly Definitions documentation.

C# System implementations must be a public static function that returns void with 1 parameter of type EcsactRuntime.SystemExecutionContext. Be very conscientious of which methods in context are called. Read more about system implementations to know what can and cannot be called.

Example


				[Ecsact.DefaultSystemImpl(typeof(example.ExampleSystem))]
				public static void AddToExampleSystem(EcsactRuntime.SystemExecutionContext context) {{'{'}}
					// Get component from the context
					var myExampleComponent = context.Get<{{'example.Example'}}>();
				{{'}'}}

Wasm System Implementations

To enable Wasm system implementations open your Unity project and go to Project Settings arrow_right Ecsact arrow_right System Implementations and change Source to WebAssembly. A .wasm file may be specified in these settings for each system. Learn more about Wasm system implementations in the Wasm Systems Overview documentation.
