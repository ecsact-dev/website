Implementing Systems in Unity

The Ecsact Unity Integration allows implementations of your declared systems.

System Declaration

Systems have a C# Attribute in its implementation. The type name of the system is put here to get the correct context. The context holds the logic of the system. This includes getting components, generating entities and retrieving data on actions. 


[Ecsact.DefaultSystemImpl(typeof(example.ExampleSystem))]
public static void AddToExampleSystem(EcsactRuntime.SystemExecutionContext context) {{'{'}}
	// Get component from the context
	var value = context.Get<{{'example.Example'}}>();
{{'}'}}

It's essential that the system is both public and static. In addition, it needs to be part of the assembly that can be generated alongside your .Ecsact file in edit -> Project Settings -> Ecsact
