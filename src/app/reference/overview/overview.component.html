Repositories Rundown

Ecsact is split into several repositories responsible for different parts of the language, runtime, and tooling.

Fundamentals

Fundamental repositories responsible for defining the Ecsact language parsing, interpretation, and the low level runtime API.

  • ecsact_runtime — C headers for the low level runtime API.
  • ecsact_parse — API for parsing Ecsact language statements.
  • ecsact_interpret — API for interpretting Ecsact language statements as runtime calls.

Language & Integration

Repositories named ecsact_lang_{{ '{' }}LANG{{ '}' }} are "language support" repositories. Integration repositories do not have a strict naming convention, but must contain the engine/framework/software name in the repository (such as ecsact_unity for the Unity game engine integration.)

  • ecsact_unity — Unity integration source code.
  • ecsact_lang_cpp — C++ related code generator plugins and C++ support libraries.
  • ecsact_lang_csharp — C# related code generator plugins and C# support libraries.
  • ecsact_lang_json — JSON related code generator plugins.

Implementation

Repositories starting with ecsact_rt_ are full or partial implementations of the Ecsact Runtime. Repositories starting with ecsact_si_ are APIs for integrating different formats as system implementations.

  • ecsact_rt_entt — Ecsact Runtime implemented with EnTT.
  • ecsact_si_wasm — API to use WebAssembly as system implementations.

Utility

  • ecsact_sdk — Source code for distributing the various binaries and the Ecsact CLI.
  • ecsact_rtb — Ecsact Runtime Builder utility command line to compile runtimes based on your Ecsact files. Mainly used in integrations.
