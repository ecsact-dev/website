Setting Up Unity

Ecsact Integration Package

Unity Ecsact Integration is provided as a Unity Package and is required to use Ecsact with Unity.

  1. Start your unity project
  2. Open the Package Manager window ( Window arrow_right Package Manager)
  3. Click the add_box in the top left
  4. Add the Ecsact Unity integration git URL 
    https://github.com/ecsact-dev/ecsact_unity.git
NOTE: You may get errors about missing components that you will need to install. Ecsact uses vswhere to output these errors, and you can search the outputs for entries.

The Ecsact Window

If the Ecsact package was successfully installed, there will be a new window under Edit arrow_right ProjectSettings arrow_right Ecsact. This is where Ecsact is configured and any setup below happens here.

Assembly

For systems to be registered into Ecsact, it needs to be part of an Assembly. In addition, it will need the following dependencies:

  1. Ecsact
  2. EcsactCSharpSystemImplRuntime

You can either do that yourself, or the integration will do that automatically with the dependencies for you.

Go to the System Implementations section of the Ecsact window and press the Create New Assembly Definitionbutton. Follow the prompt to choose where it's created.

Unity Sync

To enable Unity Sync go to the Runtime Settings section and checkmark Enable Unity Sync. When enabled, it will search for any C# files using Unity Sync automatically and display them under Unity Sync Scripts.

Recipes

Ecsact supports having multiple recipes. You can use other implementations, or there's one available for EnTT that comes packaged in the Ecsact SDK. If you're having any errors related to recipes make sure it's set under Ecsact Build Settings
