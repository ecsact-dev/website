Ecsact -> Unreal C++

A large part of what makes Ecsact work is code generation and with Unreal projects thats no exception. The Ecsact Unreal plugin is not only an Unreal plugin, but an Ecsact Codegen plugin as well!

What's Generated?

The goal of the codegen plugin is to remove a lot of boilerplate code that would otherwise be tedious to write. By default the Ecsact Unreal codegen plugin generates the following:

  • open_in_new UStructs for every component in your Ecsact file
  • open_in_new Unreal Mass Entity Fragments or Tags for every component in your Ecsact file
  • Ecsact Runner Subsystem with easily overridable component events
  • Ecsact Runner Subsystem for Ecsact to Entity Mass communication

Setup Codegen

We recommend using the EcsactUnrealCodegen command line tool that comes with the Ecsact SDK since 0.9.0. Simply execute EcsactUnrealCodegen for your project and the tool will search for all .ecsact files in your Source directory and use the Ecsact codegen tool with the Ecsat Unreal codegen plugin. The Ecsact Unreal plugin is required to be installed for the tool to find the right codegen plugin. If you run into issues please see the output of EcsactUnrealCodegen for details.

Command Line or Script

Replace path/to/project.uproject with the path to your unreal project file and path/to/unreal/install with the full path to your Unreal Engine installation directory


		EcsactUnrealCodegen path/to/project.uproject --engine-dir path/to/unreal/install

As a PreBuildStep (recommended)


		"PreBuildSteps": {{'{'}}
			"Win64": [
				"EcsactUnrealCodegen $(ProjectDir) --engine-dir $(EngineDir) --format || exit /b 1"
			]
		{{'}'}}
